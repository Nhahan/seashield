# 면접 발표 스토리라인 (PPT 구성안)

> Project SeaShield — P6 산출물. 슬라이드 골격과 각 장의 한 줄 메시지·근거 자료의
> 매핑. 발표 ~15분 + Q&A 가정, 장표 수치는 전부 레포 보고서에서 재현 가능.
> 문서 버전: v1.0 (2026-06-12)

## 0. 훅 (1장)

> **"진실을 알면 79.5%, 추정으로 쏘면 12% — 같은 탄, 같은 솔버, 5km."**

- 실험 보고서 §2의 한 행으로 시작: 무유도 요격의 본질(발사 순간의 예측이 전부)을
  숫자 하나로. "이 차이가 어디서 오는지 측정해 본 프로젝트입니다."

## 1. 문제 정의 — 요구사항의 도메인 재해석 (1-2장)

- 받은 요구: "C++, TCP/UDP 멀티클라이언트 게임서버."
- 재해석: 멀티클라이언트 = **함정 CIC의 운용석들**(지휘/사통/관측 — 기획서 §3.1) —
  요구사항을 도메인 구조로 승격. 무유도 로켓 일제사라는 "실험적 체계" 프레이밍(§2.4)
  으로 실존 체계 모사의 부담 없이 충실도를 지킨 의사결정.

## 2. 아키텍처 (2장)

- 그림 1: 2-스레드 서버(60Hz sim ↔ I/O, SPSC×2 + wakeup), Reactor(kqueue/epoll 추상화
  — Mac 개발/Linux CI), sim은 소켓을 모른다(§4.1).
- 그림 2: 3계층 크로스 플랫폼 — 서버(macOS/Linux) · `protocol/` std-only · UE5 클라가
  같은 소스를 컴파일(심볼릭 링크 단일 진실원). 클라 코어 로직(세션·보간)도 std-only로
  분리해 **헤드리스 테스트**(client-design.md §1).

## 3. 결정론 — 이 프로젝트의 척추 (2장)

- 시드(PCG32 스트림 분리) + 입력 저널 = 전체 교전 비트 재현. FNV-1a 전 상태 해시,
  (os,arch)별 골든, 리플레이 = AAR.
- **디버깅 스토리 ①**: -O2에서만 tick 300에서 골든이 어긋남 → sanitizer 무죄 입증 →
  Darwin clang의 sin+cos→__sincos 융합(1ulp) 격리 → `-fno-builtin-sin/cos`로 -O0/-O2
  비트 동일. "결정론은 선언이 아니라 회귀 테스트가 지키는 약속."

## 4. 킬체인 — 탐지·추적·사격통제 (3장)

- 레이더(정수 틱 빔 위상·Pd 시그모이드·4/3 지구 호라이즌·야코비안 R) → 칼만(Joseph
  form, NumPy 1e-9 대조, NEES/NIS) → PIP 솔버(고정점+damping). 수식↔테스트 추적표
  (simulation-models 부록 C) 한 장.
- **디버깅 스토리 ②**: 무풍인데 655m 계통 오차 → 궤적 CSV로 "예측 ToF에 탄이 PIP를
  4km 지나침" 발견 → 고각 분기의 진동 고정점 진단 → 직사탄도 재초기화 + damping으로
  5.3m. "수치해석 버그는 로그가 아니라 물리로 잡는다."

## 5. 실험 — 연구 질문에 데이터로 답하다 (3장)

- §2 거리 붕괴 곡선(진실 vs 추정), §3-4 기동의 구조적 무력화(3°/s = 격추 0, 발사 후
  위빙 = 0), §5 일제사 상관(16발 19% vs 독립 25.2%), §6 산포의 역설(발당 확률은
  그대로, 상관 파괴로 일제사 2.3배).
- 방법론 어필: 행당 시드 = 완전 재현, 비캡 would_kill 지표(리뷰 지적의 근본 해소 —
  지표 정직성), Wilson CI.
- 결론 장: "왜 현대 대공무기는 유도탄인가" — 측정으로 보인 세 가지 한계.

## 6. 성능·검증 문화 (2장)

- performance-report: 델타 압축 407→188kbps(<256 목표), 스트레스 500 엔티티 p99
  512µs(예산 8ms), 측정이 잡은 실제 이슈(미수렴 solve 60ms 스파이크 → 백오프)를
  "측정 → 발견 → 수정 → 회귀 가드"의 사례로.
- 검증 매트릭스: 테스트 213 × Debug/Rel/ASan/TSan/Docker + CI 6 jobs, 골든 회귀,
  카오스 주입, 추적성 매트릭스(traceability-matrix.md) 한 장.

## 7. 마무리 — 로드맵과 확장 (1장)

- 완료: P0~P4 + P6 헤드리스(델타·hardening·보고서). 진행: P5 UE5 클라(절차적 에셋
  파이프라인 — "물리부터 에셋까지 전부 코드").
- 확장 서사: 유도탄(PN) 2부·CIWS — "무유도의 한계 측정 → 유도의 해법" (기획서 §13의
  장기 비전과 연결).
